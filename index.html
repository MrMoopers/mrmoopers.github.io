About Me

My primary research interest is in exploring the intersection between machine learning and computer graphics. I have a particular passion for designing computer systems that allow non-expert users to create high quality and useful digital assets for use in their own industries.

Research Projects

OSMAI: A Generative OpenStreetMap Artificial Intelligence (74%)

The University of Nottingham

OSMAI

Generating maps is an essential task for various creative projects and urban development activities. Existing mapping software such as the OpenStreetMap, is often used as the starting point for creative works.

The project explored the use of Rule-Based methodologies, Generative Adversarial Networks (GANs) and Transformer models to produce synthetic map data that adheres to OpenStreetMap standards. Overall, I established a set of prompts which are able to produce viable output and I then discuss the challenges which must be considered for other research in the field.


OSMAI


Games: 2D Platformer! (65%)

The University of Nottingham

Ninja!

Creating new and different 2D platformers is difficult given the extensive library of existing games. In this project, I developed a 2D platformer with a novel gameplay element allowing the player to 'switch' between a foreground, midground and background layer using specific keys. This enabled different gameplay challenges and allowed players to use all map features to vary their gameplay. The various layers were identifiable by the colour of the terrain textures.

In addition, traps and guards were implemented to add further challenges complimenting the environmental challenge.

Mixed Reality: FishIt! (66%)

The University of Nottingham

FishIt!

Fishing is a popular recreational activity, but does have links to animal cruelty. In this project, I developed an Augmented Reality (AR) game which allowed users to simulate a fishing experience on open bodies of water. The system used the OpenStreetMap to determine the location of the user and used computer vision techniques from a mobile device's camera to determine where to overlay a virtual bobber.

Human AI Interaction Chatbot (77%)

The University of Nottingham

RPGAI

The creation of storylines for Role-Playing Games can be complex and time consuming. For this initial project using AI I developed a 'choose your own adventure' storyline which used Natural Language Processing (NLP) techniques to capture and process user input. The system predominately used Cosine Similarity to determine the dialogue tree the user was on.

The project also used sentiment analysis to aid an interaction with a Non-Player Character (NPC) witch who sought a compliment before providing the player with a boon, with catered output for any response.

FINAL YEAR PROJECT (75%)

The University of East Anglia

Dungeonify: Generating Dungeons & Dragons Battle Maps from OpenStreetMap

The generation of battle maps for role-playing games is challenging and time consuming. I created an application that generates Dungeon & Dragons battle maps from real-world scenes specified using the OpenStreetMap API. The system enables users to design complex maps just by finding a suitable location on Earth, simplifying the process (particularly for new players) and greatly reducing the time required for design, which can take many hours when done manually.

The solution enabled a user to select a number of real-world structures from the OSM and generate a correctly positioned battle map using changeable artists' textures and automatically generate a Virtual Table Top compatible file to enable its fast import into an online game. The greatest difficulty in the project was correctly orientating each structure such that all walls followed the strict 90 degree lines of the inch square grid.

The project was built in Python 3, using the Pycairo graphics library and the OpenStreetMap (OSM) API and library.


Dungeonify


Graphics 1 Game (76%)

The University of East Anglia

Racer

The design of courses for racing games can be difficult and time consuming. For this coursework project I created a 2D OpenGL C++ game which utilised procedurally generated race courses for 7 levels of difficulty. This was done through the use of 6 standardly designed track pieces, which were randomly selected and connected while being constricted to a set of rules to ensure the track was continuous and non-overlapping.

Further to this, the game featured several challenges to test player skills, including having the maps be played during the daytime, with a darkness overlay at night and in fog settings. The night and fog levels used an overlay with a headlight beam carving out a limited view of the track.

The game also featured AI drivers to race against the player and a realistic damage model for the player vehicle using a physics simulation.


Racer


THE COMPUTER SCIENCE PROJECT (95%)

The Bishop's Stortford High School

Mazer: Evaluating Algorithms for Generating & Solving Mazes in Real-Time

A project designed to compare various algorithms for generating and solving mazes visually. The .NET Framework C# Project utilises Windows Presentation Foundation (WPF) controls, which visually show the steps it completes in real-time, enabling the instant updating of visual settings while the generator or solver is running.

Algorithms were compared for running time and maze complexity. The results were automatically displayed as a table and histogram.

A total of twelve generating and solving algorithms were implemented for comparison, from the simple 'Drunkard's Walk' to the much more complex 'A*'. To ensure the solution met the requirements, it's core functionality was unit-tested

Hobby Project

The Bishop's Stortford High School

Fractal Factory: Comparing Methods for the generation of Mandelbrot Set Fractals using Multithreading.

The generation of fractals produces interesting patterns, and is a challenge to program efficently. Generating and zooming into them will require minute decimal place values used as coordinates. I experimented using generics, floats and doubles, to display the differences in the resulting image generation.

I developed this project to better my understanding of the .NET Framework and especially aid my learning of Visual Basic (Win-Forms), C# (Win-Forms) and C# (WPF).

The solution also implemented multithreading to split the generation task into four simultaneous threads, each of which updated the generating image in real-time.
